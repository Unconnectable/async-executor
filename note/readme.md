# `soml`阅读顺序：

1.  **`async-executor`** ( 必读)：**这是大脑**。了解任务是怎么被“调度”和“执行”的。逻辑最像普通写代码（队列、循环）。
2.  **`piper`** ( 推荐)：**这是血管**。类似 Go 的 Channel。代码相对独立，适合学习如何在异步任务间传递数据。
3.  **`futures-lite`** ( 基础)：**这是语法糖**。它给 `Future` 加上了 `.then()`, `.race()` 等功能。主要看它是怎么通过 Trait 扩展功能的。 
4.  **`async-io`** ( 难)：**这是感官**。这是很多初学者的噩梦。它负责和操作系统打交道（epoll/kqueue）。只看入口，别深究。
5.  **`async-fs`** ( 简单)：**这是工具**。它是 `async-io` 的具体应用，一般是用线程池模拟异步文件读写。
6.  **`smol`** ( 胶水)：**这是包装**。它没有任何逻辑，全是 `pub use`。最后看它，看它是怎么把上面几个库缝合起来的。
7.  **`async-task`** (☠️ 地狱难度)：**这是细胞**。全是 `unsafe` 和裸指针。**初学者绝对不要看源码**，知道它是做什么的就行。
